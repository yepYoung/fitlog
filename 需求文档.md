# FitLog - 极简健身饮食记录 APP

> 产品需求文档 (PRD) v1.0  
> 技术方案：PWA (Progressive Web App)  
> 目标平台：iPhone (Safari)
> 前端风格：https://github.com/Leonxlnx/liquid-glass
---

## 1. 产品概述

### 1.1 产品定位

一款面向个人使用的极简健身饮食记录应用。

**核心设计原则：快速记录优先于功能丰富，追求最少的操作步骤完成日常记录。**

### 1.2 目标用户

个人自用，无需考虑多用户、权限管理等复杂功能。

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| 极简操作 | 单次记录 < 15 秒完成 |
| 零成本部署 | PWA 方案，无需 App Store 上架 |
| 跨设备访问 | 支持云端同步（可选） |
| 数据所有权 | 数据存储在本地或自控的云端 |

App

│   ├── 今日摘要卡片

│   └── 今日记录列表

## 3. 页面设计方案

### 3.1 首页（今日概览）

#### 3.1.1 页面布局

| 区域 | 内容 | 交互 |
|------|------|------|
| 顶部栏 | 日期显示 + 问候语 | 点击日期可切换查看其他日期 |
| 摘要卡片 | 今日饮食次数、今日运动时长/目标 | 显示进度条（仅运动），直观展示完成度 |
| 快速操作 | 「+ 饮食」「+ 运动」两个按钮 | 点击进入对应记录页 |
| 今日饮食 | 饮食记录列表（按餐类分组） | 点击可编辑，左滑可删除 |
| 今日运动 | 运动记录列表（力量/有氧分开） | 显示组数×重量或时长参数 |
| 今日体重 | 体重记录卡片 | 显示当日体重，点击可记录/编辑 |

#### 3.1.2 摘要卡片设计

> 注：不追踪热量，聚焦于「吃了什么」和「练了多久」

```
┌─────────────────────────────────────┐
│  今日饮食          3 餐 + 1 加餐    │
├─────────────────────────────────────┤
│  今日运动                           │
│  45 分钟                            │
│  ████████░░░░░░░░░░░░  75%          │
└─────────────────────────────────────┘
```

### 3.2 记录页

#### 3.2.1 饮食记录表单

> 不追踪热量，仅记录「吃了什么」

| 字段 | 输入方式 | 必填 | 说明 |
|------|----------|------|------|
| 餐类 | 单选按钮组 | ✅ | 早餐 / 午餐 / 晚餐 / 加餐 |
| 食物名称 | 文本输入 + 常用列表 | ✅ | 支持从历史常用中快速选择 |
| 备注 | 文本输入 | ❌ | 可记录份量、口感等 |
| 照片 | 拍照 / 相册选择 | ❌ | 可选，图片压缩后存为 Base64 |

#### 3.2.2 运动记录表单

运动分为两大类，表单根据类别动态切换：

**A. 力量训练（Strength）**

| 字段 | 输入方式 | 必填 | 说明 |
|------|----------|------|------|
| 动作名称 | 单选 + 自定义输入 | ✅ | 卧推、深蹲、硬拉、哑铃弯举等 |
| 组数记录 | 动态列表 | ✅ | 每组记录：重量(kg) × 次数(reps)，可添加/删除组 |
| 运动时长 | 数字输入 / 计时器 | ❌ | 可手动输入分钟数或使用内置计时器，纳入每日运动统计 |
| 备注 | 文本输入 | ❌ | 如感受、调整等 |

预设力量动作（按部位分类）：

| 部位 | 动作 |
|------|------|
| 胸部 | 卧推、哑铃飞鸟、上斜卧推 |
| 背部 | 划船、引体向上、高位下拉、硬拉 |
| 肩部 | 肩推、侧平举、前平举 |
| 手臂 | 哑铃弯举、三头下压、锤式弯举 |
| 腿部 | 深蹲、腿举、腿弯举、保加利亚分腿蹲 |
| 核心 | 平板支撑、卷腹、俄罗斯转体 |

选择动作时按部位分组展示，用户可在设置中自定义各部位的动作列表

**B. 有氧运动（Cardio）**

| 字段 | 输入方式 | 必填 | 说明 |
|------|----------|------|------|
| 运动类型 | 单选 + 自定义输入 | ✅ | 爬坡、椭圆机、跑步、骑行等 |
| 时长 | 数字输入 / 计时器 | ✅ | 可手动输入或使用内置计时器 |
| 参数 | 动态字段 | ❌ | 根据类型显示：坡度(%)、速度(km/h)、阻力、距离(km) |
| 备注 | 文本输入 | ❌ | |

预设有氧运动：跑步、爬坡、椭圆机、骑行、划船机、跳绳、游泳、快走

#### 3.2.3 运动计时器

> 内置秒表功能，支持在运动过程中计时

| 功能 | 说明 |
|------|------|
| 开始/暂停 | 点击按钮开始计时，再次点击暂停 |
| 重置 | 长按或点击重置按钮归零 |
| 完成 | 点击「完成计时」自动将耗时填入时长字段（四舍五入到分钟） |
| 显示格式 | `MM:SS` 大字体显示，秒数实时跳动 |
| 后台运行 | 使用 `Date.now()` 基准，切换页面不丢失计时 |

#### 3.2.4 体重记录

> 在首页提供快速体重记录入口，用于追踪体重变化趋势

| 字段 | 输入方式 | 必填 | 说明 |
|------|----------|------|------|
| 体重 | 数字输入（小数） | ✅ | 单位 kg，支持一位小数 |
| 备注 | 文本输入 | ❌ | 如"晨起空腹"、"饭后" |

- 每天可记录多次，统计页取当日最早一条作为当日体重
- 首页显示今日最新体重和较昨日的变化值（↑/↓）
- 统计页新增体重趋势折线图

#### 3.2.5 交互要点

1. **一屏完成**：所有字段在一个页面内，不分步骤
2. **智能默认**：餐类根据当前时间自动选择
   - 7:00-10:00 → 早餐
   - 11:00-14:00 → 午餐
   - 17:00-21:00 → 晚餐
3. **运动类别切换**：顶部 Tab 切换「力量训练」和「有氧运动」
4. **快速保存**：点击保存后自动返回首页，显示 Toast 提示「已记录」
5. **常用列表**：记录 3 次以上的食物/运动自动进入常用列表

### 3.4 统计页

> 使用 Recharts 图表库实现，毛玻璃卡片风格

#### 3.4.1 页面布局

| 区域 | 内容 | 说明 |
|------|------|------|
| 顶部 | 页面标题 + 周/月切换按钮 | 切换统计维度 |
| 摘要卡片 | 运动天数、总运动时长、饮食记录数 | 3 个毛玻璃数据卡片横向排列 |
| 运动趋势 | 柱状图 | 每日运动时长（绿青渐变柱体），叠加目标线 |
| 力量训练 | 动作频次卡片 | 展示各力量动作的训练次数排行 |

#### 3.4.2 周/月切换

- 默认显示「近 7 天」
- 点击切换到「近 30 天」

#### 3.4.3 图表样式

- 柱状图圆角顶部（radius 4px）
- 运动柱：渐变 `#34D399 → #60A5FA`
- 目标线：虚线，颜色 `rgba(255,255,255,0.3)`
- 坐标轴文字：`rgba(255,255,255,0.4)`，无轴线
- 背景网格：`rgba(255,255,255,0.05)`

## 4. 用户交互流程

### 4.1 核心流程：记录一餐

```
用户打开 APP
    │
    ▼
首页：看到今日概览（1 秒了解今天状态）
    │
    ▼
点击「+ 饮食」按钮
    │
    ▼
记录页：餐类已根据时间自动选中
    │
    ├──→ 从常用列表选择食物（最快路径）
    │
    └──→ 手动输入食物名称
    │
    ▼
（可选）添加备注、拍照
    │
    ▼
点击「保存」
    │
    ▼
自动返回首页，Toast 提示「已记录」
    │
    ▼
首页刷新：今日记录列表更新，摘要数据更新

全程目标：< 15 秒完成
```

### 4.2 核心流程：查看历史

```
点击底部「历史」Tab
    │
    ▼
历史页：日历视图，有记录的日期有小点标记
    │
    ▼
点击想查看的日期
    │
    ▼
展开当天所有记录（饮食 + 运动）
    │
    ├──→ 点击某条 → 进入编辑
    │
    └──→ 左滑某条 → 删除（需确认）
```

### 4.3 辅助流程：设置

设置入口放在首页右上角，包含：

1. 每日运动时长目标（默认 60 分钟）
2. 管理常用食物列表
3. 管理常用力量动作和有氧运动列表
4. 数据导出（导出为 JSON 或 CSV）
5. 清除所有数据（需二次确认）

### 5.2 前端技术栈

| 技术 | 选型 | 说明 |
|------|------|------|
| 框架 | React 18+ | 组件化开发，生态成熟 |

### 5.3 后端方案

#### 方案 A：纯本地存储（推荐首版）

| 项目 | 说明 |
|------|------|
| 技术 | 浏览器 localStorage 或 IndexedDB |
| 优点 | 无需服务器，零成本，数据完全本地 |
| 缺点 | 换手机数据丢失，无法多设备同步 |
| 适用场景 | 个人单设备使用，不频繁换机 |
| 存储限制 | 约 5-10 MB（localStorage）或更大（IndexedDB） |

#### 方案 B：云端同步（v2.0 可选）

| 项目 | 说明 |
|------|------|
| 技术 | Supabase（后端即服务） |
| 优点 | 数据云端备份，多设备同步，换手机不丢失 |
| 缺点 | 需要网络，需注册账号 |
| 费用 | 免费套餐足够个人使用（500 MB 数据库） |
| 功能 | PostgreSQL 数据库、用户认证、实时同步 |

### 5.4 部署方案

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 代码推送到 GitHub | 版本控制 + 触发部署 |

| 5 | iPhone 安装 | Safari 打开 → 添加到主屏幕 |

> 💡 Vercel / Netlify 免费套餐完全够用，无需付费。

## 7. 设计规范

### 7.1 视觉风格 — 液态玻璃 (Liquid Glass)

> 参考实现：[liquid-glass](https://github.com/Leonxlnx/liquid-glass)  
> 灵感来源：Apple visionOS / iOS 26 液态玻璃设计语言

| 属性 | 规范 |
|------|------|
| 整体风格 | 液态玻璃（Liquid Glass），极简、现代、高级感，超越普通毛玻璃 |
| 核心技术 | `backdrop-filter: blur(24px) saturate(180%)` + 多层阴影 + 内发光边缘 |
| 背景 | 深色渐变背景（深黑→靛蓝→深紫→深蓝），多色阶过渡增加深度感 |
| 玻璃质感 | 低透明度背景（~5-6%）+ 高模糊 + 高饱和度，让内容若隐若现 |
| 内发光 | `inset` box-shadow 模拟光线在玻璃边缘折射，是液态玻璃的核心标识 |
| 顶部高光 | 卡片顶部细微高光线，模拟光源从上方照射玻璃表面 |
| 多层阴影 | 外层深度阴影 + 近距阴影 + 内边缘发光，营造悬浮立体感 |
| 留白 | 大量留白，卡片间距充足，保持呼吸感 |
| 圆角 | 卡片 22-24px 大圆角，按钮 14-16px 圆角 |
| 参考 APP | Apple visionOS、iOS 26 液态玻璃控件 |

#### 设计原则

1. **极简布局，拒绝视觉噪音**：卡片式布局区分模块，主次分明。导航集中在边缘，大面积留白展示核心内容
2. **高对比度配色，强化视觉焦点**：主色调不超过 3 种，用高饱和度色块突出关键数据
3. **字体层级清晰，信息秒懂**：核心数据用大号加粗字体，字号差异明显（数据字号 ≥ 标签字号 2 倍）
4. **图标与动效，提升交互感**：简洁图标替代文字，过渡动画增加流畅感，使用 cubic-bezier 缓动函数
5. **液态玻璃质感**：内发光边缘 + 顶部高光 + 多层阴影 = 真实玻璃光学效果，从平面到立体的升维
6. **触控反馈**：按钮按下时轻微缩放(0.96) + 阴影加深，模拟真实物理按压玻璃的手感

#### 按钮

| 类型 | 样式 | 用途 |
|------|------|------|
| 主按钮 | 渐变填充（蓝→紫）+ 内发光 + 发光阴影，白色文字，圆角 14px | 保存、确认等主要操作 |
| 次按钮 | 液态玻璃背景 + 内发光边框 + 半透明边框 | 取消、次要操作 |
| 危险按钮 | 半透明红色背景 + 红色边框 | 删除操作 |
| 快速操作 | 较大尺寸毛玻璃卡片，内含图标 + 文字 | 首页「+ 饮食」「+ 运动」 |

#### 输入框

- 高度 48px
- 圆角 14px
- 液态玻璃背景：`rgba(255, 255, 255, 0.04)` + `backdrop-filter: blur(8px)`
- 边框：`1px solid rgba(255, 255, 255, 0.08)`
- 内发光：`inset 0 0 0 0.5px rgba(255, 255, 255, 0.1)`
- 聚焦时边框变主强调色 `#60A5FA`，带发光效果 `box-shadow: 0 0 0 2px rgba(96,165,250,0.2), inset 0 0 0 0.5px rgba(96,165,250,0.3)`
- 文字白色，placeholder 半透明白色

#### 进度条

- 背景：`rgba(255, 255, 255, 0.08)` 圆角条
- 填充：渐变色（蓝→紫 / 绿→青 根据数据类型）+ 微发光效果
- 高度 6-8px，圆角与高度相同
- 发光：`box-shadow: 0 0 8px rgba(accent, 0.3)` 使进度条有光晕感

#### 底部导航

- 液态玻璃背景，`backdrop-filter: blur(28px) saturate(180%)`
- 顶部内发光线：`inset 0 0.5px 0 rgba(255,255,255,0.15)`
- 选中态图标 + 文字使用主强调色，选中图标带微发光
- 未选中态使用半透明白色
- 固定在底部，圆角 0（贴边）
