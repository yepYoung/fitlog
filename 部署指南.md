# FitLog 部署指南

> 从本地代码到线上可访问的 PWA，完整流程记录

---

## 前置条件

| 工具 | 用途 | 安装方式 |
|------|------|----------|
| Node.js | 运行构建工具 | https://nodejs.org 下载 LTS 版本 |
| Git | 版本控制 | macOS 自带，或 `xcode-select --install` |
| GitHub CLI (gh) | 命令行操作 GitHub | `brew install gh` |
| GitHub 账号 | 托管代码 | https://github.com 注册 |
| Vercel 账号 | 自动部署 | https://vercel.com 用 GitHub 登录即可 |

---

## 第一部分：本地准备

### 1.1 初始化 Git 仓库

在项目根目录下执行：

```bash
git init
```

### 1.2 创建 .gitignore

防止不必要的文件被提交到仓库：

```bash
# .gitignore 文件内容
node_modules      # 依赖包，体积大且可通过 npm install 恢复
dist              # 构建产物，Vercel 会自己构建
.DS_Store         # macOS 系统文件
*.local           # 本地环境配置
```

### 1.3 验证构建

推送前先确认代码能正常构建：

```bash
npx vite build
```

看到 `✓ built in ...` 就说明没问题。

---

## 第二部分：GitHub 操作

### 2.1 登录 GitHub CLI

```bash
gh auth login
```

按提示选择：
1. **GitHub.com**
2. **HTTPS**（协议）
3. **Yes**（用 GitHub 凭据认证 Git）
4. **Login with a web browser**（浏览器登录）

终端会显示一个一次性代码（如 `F449-7B47`），按回车后浏览器会打开，粘贴代码并确认授权即可。

> 登录状态会持久保存在本机，不需要每次都登录。

### 2.2 暂存并提交代码

```bash
# 暂存所有文件
git add -A

# 查看将要提交的文件（可选，用于确认）
git status

# 提交
git commit -m "你的提交说明"
```

### 2.3 创建远程仓库并推送

一条命令完成「创建 GitHub 仓库 + 设置 remote + 推送代码」：

```bash
gh repo create yepYoung/fitlog --public --source=. --remote=origin --push
```

参数说明：
- `yepYoung/fitlog` — GitHub 用户名/仓库名
- `--public` — 公开仓库（Vercel 免费版需要公开仓库）
- `--source=.` — 当前目录作为代码源
- `--remote=origin` — 设置远程仓库别名为 origin
- `--push` — 创建后立即推送

> 如果这条命令的 `--push` 失败了（偶尔会发生），手动执行：
> ```bash
> git push -u origin main
> ```

## 第三部分：Vercel 部署

### 3.1 首次部署（只需做一次）

1. 打开 https://vercel.com
2. 点击 **Sign Up** → 选择 **Continue with GitHub**
3. 进入 Dashboard 后点击 **Add New Project**
4. 在列表中找到 **fitlog** 仓库，点击 **Import**
5. Vercel 会自动识别为 Vite 项目，默认配置不需要修改：
   - Framework Preset: `Vite`
   - Build Command: `vite build`
   - Output Directory: `dist`
6. 点击 **Deploy**
7. 等待 1-2 分钟，部署完成后获得 URL（如 `fitlog-xxx.vercel.app`）

### 3.2 后续更新（自动完成）

以后每次修改代码，只需要：

```bash
git add -A
git commit -m "描述你改了什么"
git push
```

Vercel 会自动检测到 GitHub 上的新提交 → 自动构建 → 自动部署。通常 1-2 分钟内生效。

不需要再打开 Vercel 网站做任何操作。

---

## 第四部分：iPhone 安装 PWA

1. 用 iPhone 的 **Safari** 浏览器打开你的 Vercel URL
2. 点底部的 **分享按钮**（方框+箭头图标）
3. 选择 **添加到主屏幕**
4. 起个名字（如 FitLog），点 **添加**

这样就会在主屏幕生成一个 APP 图标，点击后全屏打开，体验接近原生 APP。

> 注意：这一步只需要做一次。后续代码更新后，APP 会自动加载最新版本（首次打开可能有缓存，关掉再开一次即可）。

## 常用命令速查

| 场景 | 命令 |
|------|------|
| 本地开发预览 | `npx vite --host` |
| 验证构建 | `npx vite build` |
| 查看改动 | `git status` / `git diff` |
| 提交并推送 | `git add -A && git commit -m "说明" && git push` |
| 查看提交历史 | `git log --oneline` |
| 查看 GitHub 登录状态 | `gh auth status` |
| 查看远程仓库 | `git remote -v` |

## 数据存储说明

- 用户数据存储在 **iPhone Safari 的 localStorage** 中（每个域名独立，上限约 5MB）
- 数据不会上传到服务器，完全保存在手机本地
- 删除方式：应用内「设置 → 清除所有数据」，或 iPhone 设置 → Safari → 网站数据
- 建议定期使用应用内「导出数据」功能备份
